Flax

2014 – now

Flax is a statically-and-strongly-typed programming language that aims to combine the power of C++ with the simplicity of C. I know, what an irony.

It is a low-level, general-purpose language offering pointer and memory manipulation, while having high-level constructs like memory-managed (through automatic reference counting) strings, generic types and functions, object-orientation, and runtime type introspection.

Currently the result of 3 years of ongoing labour, and the product of several rewrites, Flax currently possesses these features:

  • Struct and Class types
  • Function overloading
  • Parametric types and functions
  • First-class functions
  • Operator overloading
  • Traits / interfaces

The low-level interface of Flax is not hindered by an obsessive borrow checker, but it is intentionally designed not to hand-hold programmers in their work. Some deeper level of semantic analysis might be added in the future, but purely as a suggestion to the user.

Memory safety is a good thing, but it should not come at the cost of programmer productivity.

[mx]

2011 – 2016

[mx] started its life way back in 2011 with a name lost to the depths of time, as an ill-conceived foray into the field of kernel programming. The spirit of that initial leap survives in [mx] as the fifth iteration of the project.

Before abandonment due to instability, it boasted several features that would work intermittently and often mutually exclusive of each other:

  • Pre-emptive multitasking
  • Executable loading (static linking only)
  • Ring-3 userspace with homegrown libc
  • Basic self-rolled TCP/IP network stack
  • ATA reading over DMA and PIO
  • FAT32 read support

Unfortunately, I could never really fix the heisenbugs that kept cropping up, and so development lost its inertia and slowed to a halt by the end of 2016.

[mx] may be picked up again in the future, perhaps as a continuation or as a rewrite — who knows.
